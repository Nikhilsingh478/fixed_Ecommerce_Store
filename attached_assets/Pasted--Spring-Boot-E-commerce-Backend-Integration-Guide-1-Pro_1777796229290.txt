# 🛒 Spring Boot E-commerce Backend Integration Guide

---

# 1. 🧠 Project Overview

## Tech Stack
- Spring Boot 2.7.5
- Java 11
- Spring Web (REST APIs)
- Spring Data JPA / Hibernate
- Spring Validation
- MySQL (Primary DB - ORIGINAL SYSTEM)
- Embedded Tomcat

---

## Architecture

```
Controller → Service → DAO → Entity
```

- Controllers handle HTTP requests
- Services contain business logic
- DAO layer uses EntityManager (not pure repository pattern)
- Entities map to database tables

---

## High-Level Data Flow

```
Frontend → Request (Headers)
→ Controller (auth check)
→ Service
→ DAO
→ Database
→ Response (Entity/Model)
```

---

# 3. 🔐 Authentication System (CRITICAL)

⚠️ This backend does NOT use JWT.

---

## Login Endpoint

```
POST /ecommerce/login
```

---

## Required Headers

```
emailId: user@example.com
password: userpassword
```

---

## Response

### ✅ Success (200)

```json
{
  "id": 1,
  "fullName": "Test User",
  "emailId": "user@example.com",
  "userRole": {
    "id": 2,
    "userRoleName": "customer"
  },
  "password": ""
}
```

---

### ❌ Failure (403)

- Invalid credentials
- Missing headers

---

## ⚠️ Important Rules

- Every API call requires:
  ```
  emailId
  password
  ```
- No session / token system
- Fully header-based auth

# 6. 🔁 Request Patterns (IMPORTANT)

---

## Headers Used For Everything

```
emailId
password
primaryCategoryId
subCategoryId
subProductId
cartId
addressId
```

---

## ❗ Non-Standard Design

- No query params
- No RESTful URLs
- Everything passed via headers

---

## 🔄 API Flow (MANDATORY ORDER)

```
1. Login
2. Get Primary Categories
3. Get Subcategories
4. Get Products
5. Use subProductId for cart
```

# 8. ⚠️ Error Handling

---

## Status Codes

| Code | Meaning             |
|------|---------------------|
| 200  | Success             |
| 403  | Auth failure        |
| 404  | Not found           |
| 409  | Conflict/Duplicate  |

---

## Common Failures

- Missing headers
- Wrong credentials
- Missing ID headers

# 10. 🚨 Known Issues

- Header-based auth (not secure)
- No JWT
- Nested API calls (slow)
- Non-standard REST design
- Hard dependency on headers
